Updating Pages

  1. Save your changes often
  1. Do not paste content directly into the pages
  1. Remove formatting before pasting any content into the pages
  1. Prefer making changes through source code

Images

Naming an image

  1. Use lowercase letters and numbers 0-9. Don’t use punctuation or spaces. It’s best to use hyphens rather than underscores. Example: retiree-benefits.jpg
  1. Optimize file size
  1. If an image is more than 1 MB, compress it using https://tinypng.com/
  1. More from the SharePoint team - https://resources.depaul.edu/sharepoint/best-practices/image-best-practices/Pages/default.aspx

Documents (pdf, docx, pptx)

Naming files

  • Avoid spaces in filenames. Replace spaces with hyphens.
  • Avoid special characters.
  • Optional: Use all lowercase letters in filenames.
  • Don’t Use Dates or Versions. Once the file is ready to post, remove the version number. When dates or versions numbers used within published filenames, we will need to need to change filenames when those documents are updated on website.

Workflow when numbers/versions are used in filenames

  1. Upload a new file every time when the file needs to be replaced.
  1. Change the link to the file.
  1. Update all links across HR webpages.
  1. Send approval request to publish the page.

Workflow when numbers/versions are NOT used in filenames (preferred)

  1. Simply replace the old file with the new one.
  1. Does not require any page approvals

Uploading

  1. Before uploading a file, check for duplicates. We don’t want to have two versions of the same file in multiple locations.
  1. Always check the document folder of the Forms page for any duplicates

    https://offices.depaul.edu/human-resources/forms/Pages/default.aspx

Approval requests

  1. All updates made on the front-end requires approval. Example: Content update, image update, link update.
  1. All updates made on the back-end do NOT require any approval. Example: REPLACING a file in the file directory.
  1. No requests needed to delete a page. Publishers can unpublish a page.

Changing URL

Always get assistance from the Marketing communications team before changing a url.

General workflow is

  1. Duplicate page to a new url
  1. Convert page with old link to a redirect page

    https://resources.depaul.edu/sharepoint/page-layouts/specialty-pages/Pages/Redirect-Page.aspx

  1. Set redirection link to the new page
  1. Delete the old page after 90 days
